  • <static> {Boolean} puredom.net.multiLoad(resources, callback)
    Make multiple HTTP requests in order, firing the callback only when all have completed.
    Callback Format:
    	callback(
		success   // {Boolean} - did *any* requests succeed?
		responses // {Array}   - responses corresponding to the provided resources.
		successes // {Number}  - how many requests succeeded (status<400)
		failures  // {Number}  - how many requests failed (status>=400)
	);
    Parameters:
    {Object[]} resources
    An array of resource objects, with format as described in puredom.net.request options.
    {Function} callback Optional
    A function to call once all requests have completed, with signature function(success, responses, successes, failures). [See description]
    Returns:
    {Boolean} returns false if no resources were provided.
  • <static> {puredom.net.HttpRequest} puredom.net.post(url, post, callback, options)
    Make an HTTP POST request. This is a convenience wrapper around puredom.net.request.
    Post value type conversion:
    Object: Objects get automatically converted to a querystring-encoded Strings through puredom.querystring.stringify.
    String: Strings are used as the POST body, without any conversion. 
    	puredom.net.get("/ajax?f=2", { foo:'bar' }, function(s, data) {
		console.log(s===true, data);
	});
    Parameters:
    {String} url
    The URL to which a request should be sent.
    {Object|String} post
    POST body (see description). If this value is set, the request type will be POST unless overridden via options.type.
    {Function} callback
    A function to be called when the request has completed, with signature: function({Boolean} success, {Object|String|Document} data)
    {Object} options Optional
    Additional configuration. See puredom.net.request.
    Returns:
    {puredom.net.HttpRequest} An HTTP request object
  • <static> {puredom.net.HttpRequest} puredom.net.request(options, callback)
    Construct and send an HTTP request based on a configuration object (options).
    Options:
    {String}urlRequired. A URL to which the request should be sent.
    {String}typeAn explicit request type (HTTP verb). Generally "GET" or "POST". Overrides other methods of setting request type.
    {Object|String}postPost body. {Object}s are converted to a form-encoded body using puredom.parameterize. {String}s are used as the POST body with no manipulation. If this value is set, the request type will be POST unless overridden by *type*.
    {Function}callbackA function to be called when the request has completed, with signature: function({Boolean} success, {Object|String|Document} data)
    {Object}headersHashmap of request headers. (key-value)
    {String}contentTypeOverrideIf set, overrides the *Content-Type* header returned by the server.
    Parameters:
    {Object} options
    Define request options
    {Function} callback Optional
    A callback function, used if options.callback is not set.
    Returns:
    {puredom.net.HttpRequest} An HTTP request object
